# PLAN — GOTY Phase 6: 깊이 팩 (말씀 새기기 · 긴 밤의 빈 그물)

## 문제

사용자 피드백: **"애들이 너무 쉽게 끝내버려."** 원인 진단: 카드를 읽지 않고
"계속 걷기"만 눌러도 14곳이 전부 새겨진다 — 참여 없는 완주가 가능하다.

## 해법 원칙

벌점형 난이도가 아니라 **참여 강제**: 장소가 지도에 새겨지려면 그 장소의
말씀 한 구절을 스스로 완성해야 한다. 실패해도 벌은 없다 — 다시 고르면 된다.
읽기가 곧 메커니즘이 된다. (게임의 기존 문법: "메커니즘이 곧 의미다.")

성구 출처 표기는 기존 verse 필드의 괄호(눅 5:10 등)를 재사용해 빈칸 UI 아래
작게 표시.

### UI (index.html + styles.css)
카드 안, artifact 아래에:
```html
<div id="card-seal" class="hidden">
  <p class="seal-head">말씀을 새기라</p>
  <p id="seal-verse"></p>      <!-- 앞부분 + <span class="seal-blank">＿＿＿</span> + 뒷부분 -->
  <div id="seal-chips"></div>  <!-- 버튼 3개, 셔플 -->
  <p id="seal-ref" class="dates"></p>
</div>
```
- 스타일: 기존 verse 박스(베이지)와 한 계열이되 금박 테두리, chips는 최소
  높이 44px(모바일 탭), 양피지 버튼 언어 재사용.
- 정답: 빈칸이 금색 잉크로 채워지는 트랜지션(0.4s) + 기존 chime 재생 +
  "다녀감" 스탬프 등장 → 닫기 버튼 활성.
- 오답: 칩이 0.3s 흔들리고(shake keyframes) 1s 동안 dim, 구절 박스도 살짝
  흔들림. 재시도 무제한, 카운터 없음.
- `prefers-reduced-motion`: 흔들림 대신 칩 테두리만 붉게 1s.

### 로직 (js/main.js)
1. **차팅 시점 이동**: 현재 첫 방문 카드 열기에서 실행되는 차팅(fillRelic·
   visitedCount++·persist — 정확한 함수는 grep으로: `chartSite` 호출 경로)을
   **씰 완성 시점**으로 옮긴다. 카드 열기 자체는 차팅하지 않는다.
2. 첫 방문(미차팅) 카드: `card-seal` 표시, 닫기 버튼은 비활성 + 라벨
   "말씀을 새겨야 지도에 남는다". 씰 완성 → 차팅 실행 → 닫기 버튼
   "계속 걷기" 활성. **씰 미완성 상태로 카드를 닫는 다른 경로가 있으면
   전부 차단**(오버레이 바깥 탭 등 — grep으로 확인).
3. 재방문(차팅됨): seal 박스 숨김, 기존 그대로.
4. 씬이 있는 사이트(nets 대화·고백·부인·회복·물 위 걷기·항해 등): 씬은
   기존대로 카드 **앞에서** 진행, 씰은 카드 안에서 동일하게.
5. 에필로그/피날레 트리거(visitedCount===14)는 차팅 시점이 옮겨졌으므로
   자동으로 "14번째 씰 완성" 순간이 된다 — 코드 변경 불필요함을 확인만.
6. 칩 셔플: Math.random 3개 순서 섞기(런타임이므로 허용), 텍스트는
   textContent로만 주입(HTML 인젝션 방지).
7. 저장 스키마 무변경 (charted가 곧 "씰 완성"의 기록이 된다).

## B. 긴 밤 — 빈 그물 세 번 (long-night 미니 비트)

현재 long-night는 밤 배 위에서 카드가 바로 열린다. 카드가 열리기 **전에**:
1. 대화 UI 재사용(showDialog/dialogButton — 정확한 시그니처 grep):
   화자 없음, 버튼 "그물을 던진다".
2. 3회 반복: 탭 → splash 오디오 + 2.2s 대기(그물 올리는 사이) → 자막:
   1회차 "아무것도 없다.", 2회차 "밤이 깊어간다. 여전히 아무것도.",
   3회차 "동이 트는데도, 그물은 비어 있다."
3. 3회째 자막 후 1.5s 뒤 카드 오픈(씰 포함). 중도 이탈 불가(모달).
4. 이미 차팅된 재방문에는 미니 비트 생략.

## C. 인도자 가이드 (guide.html)

"말씀 새기기" 안내 한 단락: 이제 각 장소는 말씀 한 구절을 완성해야 지도에
남는다는 것, 틀려도 불이익이 없으니 아이들이 본문을 다시 읽게 하라는 것.
기존 가이드 문체(존댓말) 유지.

## 제약

- 벌점·타이머·점수 금지 — 마찰은 오직 "읽어야 한다"뿐.
- 기존 씬·오디오·차팅 저장 포맷 무변경. 세이브 마이그레이션 불필요.
- 씰 문안은 위 표를 **글자 그대로** — 임의 수정 금지 (성구 정확성).
- 모든 새 텍스트는 textContent 주입.
- 주석 한국어 문체.

## 검증 (구현자 셀프체크)

1. node --check + 콘솔 0.
2. 프레시 세이브 → nets 방문: 씬 → 카드 → 닫기 비활성 확인 → 오답 탭(흔들림,
   차팅 안 됨) → 정답 탭(금색 채움 + chime + 스탬프 + 닫기 활성) → HUD
   "1 / 14" 확인.
3. 카드 재오픈: 씰 없음.
4. long-night: 세 번의 빈 그물 → 카드 → 씰. 재방문 시 비트 생략.
5. localStorage에 charted 반영이 **씰 후**에만 일어나는 것 확인.
6. 씰 미완성으로 카드를 닫는 우회 경로 전무 확인 (Esc, 바깥 탭, M 등).
7. 모바일 뷰포트: 칩 3개 한 줄 or 줄바꿈 자연스러움, 44px 탭.
8. 14곳 전부의 seal 데이터 렌더 스모크(콘솔 루프로 데이터 검증 가능).
