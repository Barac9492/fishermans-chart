## PLAN DEVIATION 목록
1. **Sky scale 4000→1200**: 카메라 far가 1600이라 4000 박스는 잘린다.
   1200(모서리 ~1039)로 far 안에 넣고 플레이어 추종.
2. **PMREM 128 해상도 → 내부 기본 256**: `PMREMGenerator.fromScene`은 큐브
   해상도 공개 API가 없다. 베이크 씬이 박스 1개라 3장 합쳐도 체감 비용 없음
   — 전체 페이지 로드 718ms (스크립트+베이크 포함), +300ms 예산 안.
3. **반사 수면 y -0.42→-0.14**: 바다 평면(호수 밑까지 깔림)의 정점 파도
   마루가 -0.18까지 올라와 거울면을 뚫었다. 시각 전용 평면이라 판정 무관.
4. **돔 재질: 석회암 아닌 금 유지**: 계획 ④-2의 "Standard(석회암)"를 돔에
   그대로 적용하면 피날레("돔을 돌며")의 금 돔 정체성이 사라진다. 돔은
   Standard(goldBright, metalness 0.85, roughness 0.3, emissive 유지)로 —
   PMREM 반응하는 진짜 금속. 석회암 0.7은 드럼에 적용.
5. **③④는 세 모드 공통** (계획 서두는 "lite: G5 전부 OFF"): 지오메트리/재질
   스왑을 모드별로 되돌리려면 구·신 지오메트리 이중 보관이 필요한데, lathe
   24세그·노멀맵 1장의 비용은 사실상 0이라 상시 적용했다. 성능이 실제로
   걸리는 ①(하늘+환경맵)과 ②(반사 2패스)만 게이트. lite의 하늘·물·포스트는
   이전과 동일 경로.
6. **여밈선/트림 수치 재정렬** — 계획이 "재정렬"만 명시, 구체 수치(0.68 /
   z-0.47 / 0.48)는 실측으로 정했다.

- 확인: 갈릴리 한낮(w 0.85) — 백색 헤이즈 대신 지평선 웜톤→천정 파랑의
  따뜻한 아침 하늘. 어스름(w 0.37, warmthOverride로 강제): 해 쪽 지평선
  복숭아빛 노을 그라데이션 그대로 — 회귀 없음 (rayleigh 3.2 경로 무변경).
- 능선 스프라이트: 파래진 하늘 위에서 윗변 하드 엣지 육안 재검 — 부드럽게
  섞여 들어감, 추가 페이드 불필요 (기존 좌우 40px 페이드만으로 충분).

### 검증 절차
새 탭에서 localStorage 클리어 → 리로드 → 새 게임. 검증용 임시 훅(__g5t:
텔레포트·카메라·warmthOverride)을 썼다가 **완전 제거 후** 최종 리로드로
재확인 (파일 내 훅 0건, 콘솔 오류 0). 스크린샷: 정면·후면 클로즈업,
기본 카메라 원경, 앉기, 갈릴리 한낮 하늘, 어스름 하늘. 임시 스크린샷
파일은 삭제 — 트리 클린.

### 솔직한 실루엣 판정
이제 "아이"가 아니라 "망토를 걸친 성인 순례자"로 읽힌다. 곧게 떨어지는
로브 + 전신 길이의 청록 덧옷 + 드러난 얼굴이 결정적이었다. 머리는 여전히
스타일라이즈드하게 큰 편(약 4등신)이지만 이 게임의 목각 인형 미학 안에서는
자연스럽다 — 사실적 7등신을 원한다면 player 스케일 체계(비주얼 y만 늘리기)를
따로 손대야 하며 이는 이번 범위 밖.
