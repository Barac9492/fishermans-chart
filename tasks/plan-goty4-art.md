그래픽을 "깔끔한 로우폴리"에서 "의도된 화풍"으로. 네 기둥: ① 지역 색
스크립팅 ② 베드로 실루엣 ③ 물가(깊이·포말) ④ 지평선. 전부 코드 생성

프레임당 신규 연산은 스카이라인 위치 추종 + 포말 불투명도뿐. fpsGate 무관.

## 변경 명세 (전부 js/main.js)

### ① 지역 색 스크립팅 — 오버레이 한 장
걷는 땅은 평평하게 두고, **색만** 지역을 따라 흐르게 한다.

1. `makeRegionTint()`: 256×768 캔버스. 월드 z -205→163, x -58→58을 세로로
   매핑해 그린다:
   - 갈릴리(z<-95): 물가(LAKE 폴리곤 주변 ~8유닛)는 초록기 도는 모래
     `rgba(150,160,110,α)`, 그 밖은 따뜻한 모래 (α는 12-20% 수준 — 은은하게)
   - 요단 길(-95<z<88): 중앙 스파인(x±6)을 따라 붉은 황토 띠, 가장자리로 소멸
   - 예루살렘(z>88): 차가운 석회암 `rgba(190,195,205,α)` 워시
   - 새벽 바닷가(두 번째 불 주변, z≈-120 서안): 장밋빛 금 워시 소량
   - 전체에 큰 스케일의 명암 얼룩 4-6개(라디얼)로 단조로움을 깬다.
2. 적용: MAIN_LAND extrude와 **같은 Shape**로 ShapeGeometry(윗면만) 하나 더
   생성, y=0.02, `MeshLambertMaterial({ map: tintTex, transparent: true,
   depthWrite: false })` + `polygonOffset` — 월드→UV 매핑은 ExtrudeGeometry
   UV 규칙(도형 좌표 그대로)이라 `repeat.set(1/116, 1/368)` + offset으로 맞춘다.
   ※ 구현자: UV 방향(v 반전) 실측으로 확인 — 갈릴리 초록이 북쪽(-z)에
   와야 한다. 그림자 receiveShadow true.
3. 로마 땅은 손대지 않는다 (한 장으로 끝).

### ② 베드로 실루엣 (1159-1195 그룹에 추가/수정)
뒤태(클록 콘)가 화면의 90%다 — 뒤에서 읽히는 디테일에 집중:
1. **두건(mantle)**: 머리 뒤-어깨를 덮는 반구 셸 `SphereGeometry(0.5, 10, 6,
   0, π*2, 0, π/2)` 살짝 뒤로 기울여 클록 위 목 뒤에 — 클록보다 한 톤 밝은
   같은 계열 (0x6b7e78).
2. **클록 밑단 트림**: 얇은 토러스(반지름 0.62, 튜브 0.03) 클록 하단 —
   COLORS.woodDark. 클록에 세로 여밈선: 가는 박스(0.03×1.2×0.03) 등 중앙.
3. **손**: 팔 박스 끝에 살구색 스피어(0.09) 두 개 — 걸을 때 팔과 함께
   회전하도록 armL/armR 메시에 child로 붙인다 (그룹 재구성 금지, 기존
   애니 코드가 armL.rotation.x를 직접 쓰므로 자식이면 자동 추종).
4. **머리채**: 머리 뒤통수에 어두운 반구 캡(수염색 0x4a4038) — 뒤에서 볼 때
   맨들한 공이 아니게.
5. **눈**: 정면에 작은 다크 스피어 2개(0.045) — 카드/컷신에서 얼굴이 돌 때
   생기를 준다.
6. 발: 기존 다리 박스 하단에 샌들 박스(0.2×0.06×0.3, woodDark) — legL/legR의
   child로 (스윙 자동 추종).
※ 그림자 castShadow는 두건·트림만 true (소품은 생략 — 드로콜 예산).

### ③ 물가 — 깊이 그라디언트 + 포말선
1. **정점 색**: lakeWater·seaWater 재질에 `vertexColors: true` 추가하고
   빌드 타임에 각 정점의 물가 거리로 색을 채운다 — 물가 0-6유닛은 얕은
   청록(밝게 +20%), 깊은 곳은 기존색. 거리 계산은 폴리곤(LAKE, MAIN_LAND
   해안) 선분까지 최단거리 — 한 번만 계산 (71×61 + 21×21 정점, 빌드 타임
   OK). 재질 color는 0xffffff로 바꾸고 정점색에 원래 색을 굽는다
   (color×vertexColor 이중 적용 방지).
2. **포말선**: LAKE 폴리곤을 따라 폭 0.7의 리본 스트립(인접 점 잇는 사각형
   띠, 나이브 마이터). `MeshBasicMaterial({ color: 0xfffaf0, transparent,
   opacity 0.25, depthWrite: false })`, y=0.03. animate에서
   `opacity = 0.16 + 0.1 * sin(t*0.9)` 숨쉬기. 바다 쪽은 갈릴리 서안
   (두 번째 불 해변 근처 MAIN_LAND 세그먼트 z -160~-104 구간)만 — 전체 해안
   리본은 예산 초과.
3. 기존 파도 루프(4294-4300)는 무변경.

### ④ 지평선 — 스카이라인 링 + 해 무리
1. `makeRidgeSprite(seed, w, h, color)`: 캔버스에 능선 실루엣(랜덤워크
   봉우리) 1장 → THREE.Sprite (`fog: false`인 SpriteMaterial, transparent).
2. 4장: 북(골란 고원, 높음)·동(모압 산지)·서(지중해 쪽 낮은 언덕)·남서.
   해/달 스프라이트처럼 **매 프레임 player 기준 재배치**(3865-3885 블록에
   합류): `pos = player + dir*700`, y는 능선 하단이 시야 지평선(≈-20)에
   오게. scale ~(900, 140).
3. 색·불투명도: 밤엔 진한 남색 실루엣, 낮엔 옅은 흙빛 헤이즈 —
   `opacity = chartUp ? 0 : 0.5 - 0.25*dayK`, 색은 dayK로 lerp (기존 dayK/
   nightK 변수 재사용). 차트 뷰에서는 숨김 (기존 패턴).
4. **해 무리(halo)**: sunSprite 뒤에 라디얼 그라디언트 스프라이트 1장
   (additive, scale 3배), `opacity = 0.35 * dayK` — sunSprite 재배치 코드에
   한 줄 합류.

## 제약

- 걷는 땅의 지오메트리(높이·폴리곤)와 isWalkable/충돌 코드는 절대 불변.
- 기존 재질/색 상수 재사용 (COLORS), 새 상수는 COLORS에 추가.
- 캐릭터 그룹 재구성 금지 — 기존 애니 참조(legL 등)를 깨지 않게 child로만.
- 파도 루프·PMREM·bloom 등 렌더 패스 산출물 무변경.
- 주석은 기존 한국어 문체.
- 모든 신규 요소는 chart 뷰(M)에서 어색하지 않아야 함: 오버레이는 보임(OK),
  스카이라인·해 무리는 opacity 0.

## 검증 (구현자 셀프체크)

1. `node --check`(mjs 복사) + 콘솔 오류 0.
2. 스크린샷 3장: (a) 갈릴리 물가 — 초록기 모래·포말선·얕은물 밝음,
   (b) 베드로 클로즈업(뒤·앞) — 두건·트림·손·눈, (c) 지평선 — 능선 실루엣과
   해 무리. 밤(예루살렘) 1장 — 능선이 실루엣으로 어두워지는지.
3. M(차트 뷰) 진입 — 스카이라인/해 무리 사라짐, 오버레이 색이 지도에서
   지역감을 줌, z-fight 없음.
4. 이동·상호작용 회귀 없음 (걷기, E 방문, 물 위 걷기 진입 지점만 육안).
5. FPS: 시작 4초 게이트 통과(usePost 유지) 확인.
